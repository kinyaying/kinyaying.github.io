从零搭建vue脚手架

本文总阅读量次

配置webpack

  • 1、安装webpack
    为了避免webpack的版本差异，最好选择本地安装，而不是全局安装。由于webpack上线的时候就不再需要，所以保存成开发依赖即可（–save-dev）。

  • 2、配置npm脚本
    在package.json中配置scripts字段来定义脚本命令，这样当在终端上执行npm start，会在当前目录下找webpack.config.js文件，并执行webpack-dev-server --port 8080。webpack-dev-server是一个小型的node.js的express服务，--port是端口号的指向。而在终端上运行npm run build，则是使用webpack的打包功能，目标文件夹(webpack.config.js中会配置)下会出现打包后的文件。

  • 3、配置webpack.config.js
    根目录下新建一个文件，文件名为webpack.config.js，这就是webpack的配置文件。编辑这个文件，添加entry和output，并导出配置项。entry是指项目的入口，一般是src目录下的mian.js文件。webpack会自动查找相关的依赖，进行打包。output是指打包后文件的相关配置，其中path是打包后文件的存放路径，此路径必须是绝对路径(webpack API的规定)，所以需要使用path这个内置模块将相对路径解析出绝对路径。filename是打包后的文件名。

设置完配置项后，新建src目录，并添加main.js文件，此文件必须和配置里的entry字段相一致，否则webpack找不到入口文件会报错。

数据层vuex

视图层的工作弄完了，我们再搭建一下数据层。

vue里面的数据传递可以通过找共同的父组件通过属性一层层的传递来进行。
另一种方式可以通过eventbus来传递。但是eventbus的本质是事件的发布订阅模式，比较混乱，如果项目大了，需要监听多个事件，还可能造成命名冲突等问题。
vuex解决可以解决平级组件以及跨组件的数据传递问题。vuex主要用在大型项目中，来将数据进行统一管理。

添加nodejs模块

其实这个部分已经不属于脚手架的部分了，只是在开发的过程中，前端和后台的开发工作会同时进行。所以前端需要先利用假数据进行开发，完成后再跟后台进行数据联调。这时就需要我们在本地利用nodejs搭建一个本地的服务，读取假数据，返回给前端。
vuex不能直接修改状态，虽然这么做是不会报错的，但是
vuex的理念是单项数据流，不允许组件直接修改状态

更多的思考

虽然脚手架搭建完毕了，但是还有些可以优化的地方。
1、将某些公用的组件变成vue的全局插件
其实，像loading、message(消息框) alert这种具有通用性的组件，可能多个页面都需要用到。如果我们在每个页面都挂载这些组件，就会增加一些多余的重复代码。所以实现全局的组件是一个不错的选择。
那么我就以loading为例，把它挂载到全局组件上。

2、当然我们还可以结合原生APP做其他优化。比如如果我们的h5的包是以hybird的形式嵌入到一个原生App中的话，我们可以考虑将Navbar变成原生的组件。也就是说在浏览器里顶部导航条是h5导航条，而在手机里顶部的导航条是原生的，下面的容器才是webview浏览器。这种方式有一个好处，就是如果h5页面内部逻辑出错，导致页面白板，至少能够保证Navbar上的退出按钮是可用的。用户可以进行退出操作，不用强制把应用关闭。(这可是我在项目中亲身经历过的，说多了都是泪~)。

3、其实我上面的webpack的配置还是很粗糙的，只是实现的基本的配置，还可以进一步的优化。
